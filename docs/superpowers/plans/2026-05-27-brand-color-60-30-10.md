# 60-30-10 Brand Color Refresh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the app's muted primary color `#A85A1F` with the sampled logo orange `#FF6E11`, redistribute color via the 60-30-10 rule with orange as the 10% accent, and clear WCAG AA contrast across all chrome and text pairs.

**Architecture:** All color decisions flow through [lib/src/shared/widgets/app_theme.dart](../../../lib/src/shared/widgets/app_theme.dart). Update palette constants, `ColorScheme.fromSeed`, and component themes there; then migrate the 14 callsites of the retired `amuwakSoftAccent` token to inline orange-tinted substitutions; then fix two screens that hand-roll brand colors (login brand mark, dashboard hero header).

**Tech Stack:** Flutter, Material 3, flutter_test for widget tests.

**Spec:** [docs/superpowers/specs/2026-05-27-60-30-10-brand-color-design.md](../specs/2026-05-27-60-30-10-brand-color-design.md)

**Substitution rules** for the deleted `amuwakSoftAccent` token (used in 14 places across 6 files):

- `Border.all(color: amuwakSoftAccent)` → `Border.all(color: amuwakPrimary.withValues(alpha: 0.18))` — hairline orange-tinted card/button outline
- `color: amuwakSoftAccent` (inside `BoxDecoration` of icon backdrop containers) → `color: amuwakPrimary.withValues(alpha: 0.12)` — orange-tinted icon backdrop chip

These substitutions create the "orange-as-10%" pattern across the app without introducing a new theme token.

**Test execution note** (from project memory): On this Windows host, `flutter test path1 path2 ...` hangs at loading. Always run **one file at a time**: `flutter test test/path/file_test.dart`.
